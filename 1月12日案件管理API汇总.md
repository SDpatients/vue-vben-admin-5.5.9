## 通用说明

### 基础URL

```
http://your-domain/api/v1
```

### 请求头

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Content-Type | String | 是 | application/json |
| Authorization | String | 是 | Bearer {token} |

### 通用响应格式

所有接口返回统一的JSON格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 2. 更新案件信息

### 接口信息

- **请求路径**: `/case/{caseId}`
- **请求方法**: `PUT`
- **接口描述**: 更新案件基本信息

### 请求参数

#### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| caseId | Long | 是 | 案件ID |

#### 请求体参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| caseName | String | 否 | 案件名称 |
| caseReason | String | 否 | 案件原因 |
| remarks | String | 否 | 备注信息 |
| filingDate | LocalDate | 否 | 立案日期 |
| caseProgress | String | 否 | 案件进度 |
| mainResponsiblePerson | String | 否 | 主要负责人 |
| designatedInstitution | String | 否 | 指定机构 |
| acceptanceCourt | String | 否 | 受理法院 |
| debtClaimDeadline | LocalDateTime | 否 | 债权申报截止时间 |

#### 请求示例

```http
PUT /api/v1/case/123
Content-Type: application/json

{
  "caseName": "某公司破产清算案(更新)",
  "caseReason": "资不抵债，无力偿还",
  "remarks": "更新备注信息",
  "filingDate": "2024-01-10",
  "caseProgress": "SECOND"
}
```

### 成功响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

### 错误响应示例

```json
{
  "code": 400,
  "message": "案件不存在或参数错误",
  "data": null
}
```

## 3. 案件状态流转

- **请求路径**: `/case/{caseId}/status`

- **接口描述**: 更新案件状态

### 请求参数

#### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| caseId | Long | 是 | 案件ID |

#### 请求体参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| caseStatus | String | 是 | 案件状态 |

| 值 | 说明 |

```http
PUT /api/v1/case/123/status

}
```

```json
{
  "code": 200,
```

### 错误响应示例

## 4. 案件进度更新

### 接口信息

- **请求路径**: `/case/{caseId}/progress`
- **请求方法**: `PUT`
- **接口描述**: 更新案件进度信息

### 请求参数

#### 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| caseId | Long | 是 | 案件ID |

#### 请求体参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| stageName | String | 否 | 阶段名称 |
| stageDescription | String | 否 | 阶段描述 |
| startDate | LocalDate | 否 | 开始日期 |
| endDate | LocalDate | 否 | 结束日期 |
| expectedEndDate | LocalDate | 否 | 预计结束日期 |
| progressStatus | String | 否 | 进度状态 |
| completionPercentage | Integer | 否 | 完成百分比(0-100) |
| keyTasks | String | 否 | 关键任务 |
| completedTasks | String | 否 | 已完成任务 |
| pendingTasks | String | 否 | 待完成任务 |
| issues | String | 否 | 存在问题 |
| solutions | String | 否 | 解决方案 |
| attachments | String | 否 | 附件 |
| responsiblePerson | String | 否 | 负责人 |
| responsiblePersonId | Long | 否 | 负责人ID |
| remarks | String | 否 | 备注 |
| isCompleted | Boolean | 否 | 是否完成 |

#### 请求示例

```http
PUT /api/v1/case/123/progress
Content-Type: application/json

{
  "stageName": "债权申报阶段",
  "stageDescription": "收集和审核债权人申报的债权",
  "startDate": "2024-01-08",
  "expectedEndDate": "2024-03-08",
  "completionPercentage": 50,
  "keyTasks": "债权申报、债权审核、债权确认",
  "completedTasks": "债权申报",
  "pendingTasks": "债权审核、债权确认",
  "responsiblePerson": "张三",
  "isCompleted": false
}
```

### 成功响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

### 错误响应示例

```json
{
  "code": 400,
  "message": "完成百分比不能小于0或大于100",
  "data": null
}
```

## 5. 创建案件

### 接口信息

- **请求路径**: `/case`
- **请求方法**: `POST`
- **接口描述**: 创建新案件

### 请求参数

#### 请求体参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| caseNumber | String | 是 | 案号 |
| caseName | String | 是 | 案件名称 |
| acceptanceDate | LocalDate | 是 | 受理日期 |
| caseSource | String | 否 | 案件来源 |
| acceptanceCourt | String | 否 | 受理法院 |
| designatedInstitution | String | 否 | 指定机构 |
| mainResponsiblePerson | String | 否 | 主要负责人 |
| isSimplifiedTrial | Integer | 否 | 是否简易程序(0否,1是) |
| caseReason | String | 否 | 案件原因 |
| caseProgress | String | 否 | 案件进度(默认FIRST) |
| debtClaimDeadline | LocalDateTime | 否 | 债权申报截止时间 |
| filingDate | LocalDate | 否 | 立案日期 |
| remarks | String | 否 | 备注信息 |

#### 案件进度枚举值

| 值 | 说明 |
|----|------|
| FIRST | 第一阶段 |
| SECOND | 第二阶段 |
| THIRD | 第三阶段 |
| FOURTH | 第四阶段 |
| FIFTH | 第五阶段 |
| SIXTH | 第六阶段 |
| SEVENTH | 第七阶段 |

#### 请求示例

```http
POST /api/v1/case
Content-Type: application/json

{
  "caseNumber": "破(2024)02号",
  "caseName": "某公司破产清算案",
  "acceptanceDate": "2024-01-12",
  "caseSource": "法院指定",
  "acceptanceCourt": "北京市第一中级人民法院",
  "designatedInstitution": "某律师事务所",
  "mainResponsiblePerson": "张三",
  "isSimplifiedTrial": 0,
  "caseReason": "资不抵债",
  "caseProgress": "FIRST",
  "debtClaimDeadline": "2024-03-12 12:00:00",
  "filingDate": "2024-01-12",
  "remarks": "新创建案件"
}
```

### 成功响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "caseId": 124,
    "caseNumber": "破(2024)02号"
  }
}
```

### 错误响应示例

```json
{
  "code": 400,
  "message": "案号不能为空",
  "data": null
}
```

## 8. 案件简单信息查询(分页)

### 接口信息

- **请求路径**: `/case/simple-list`
- **请求方法**: `GET`
- **接口描述**: 查询案件的简单信息列表，支持分页

### 请求参数

#### 查询参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|

| size | Integer | 否 | 每页大小(默认10) |
| caseNumber | String | 否 | 案号(模糊查询) |

#### 请求示例

```http
GET /api/v1/case/simple-list?page=1&size=10&caseNumber=破(2024)
```

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "list": [
      {
        "id": 123,
        "caseNumber": "破(2024)01号",
        "caseName": "某公司破产清算案"
      },
      {
        "id": 124,
        "caseNumber": "破(2024)02号",
        "caseName": "某公司破产重整案"

  }
}
```

## 附录

### 枚举值汇总

| 值 | 说明 |
|----|------|
| PENDING | 待处理 |
| IN_PROGRESS | 进行中 |
| COMPLETED | 已完成 |
| CLOSED | 已结案 |
| TERMINATED | 已终结 |
| ARCHIVED | 已归档 |

#### 案件进度 (CaseProgress)

| 值 | 说明 |
|----|------|
| FIRST | 第一阶段 |
| SECOND | 第二阶段 |
| THIRD | 第三阶段 |
| FOURTH | 第四阶段 |
| FIFTH | 第五阶段 |
| SIXTH | 第六阶段 |
| SEVENTH | 第七阶段 |

#### 审核状态 (ReviewStatus)

| 值 | 说明 |
|----|------|
| PENDING | 待审核 |
| APPROVED | 已通过 |
| REJECTED | 已驳回 |

### 数据类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| Long | 长整型 | 123 |
| Integer | 整型 | 10 |
| String | 字符串 | "破(2024)01号" |
