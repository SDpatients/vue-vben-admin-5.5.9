## 1. 接口概述

### 1.1 接口列表

| 序号 | 接口名称 | 请求方式 | 接口路径 | 说明 |
|------|----------|----------|-----------|------|
| 1 | 获取归档分类树 | GET | /archive/categories | 获取归档分类的完整树形结构 |
| 2 | 上传归档文件 | POST | /archive/{caseId}/upload | 上传归档文件到指定案件的指定分类 |

| 5 | 更新归档记录 | PUT | /archive/record/{recordId} | 更新归档记录信息 |
| 6 | 删除归档文件 | DELETE | /archive/record/{recordId} | 删除归档文件(逻辑删除) |
| 7 | 批量删除归档文件 | DELETE | /archive/records/batch | 批量删除归档文件(逻辑删除) |
| 8 | 获取归档文件统计 | GET | /archive/{caseId}/statistics | 获取指定案件的归档文件统计信息 |
| 9 | 下载归档文件 | GET | /archive/file/{fileId}/download | 下载归档文件 |
| 10 | 预览归档文件 | GET | /archive/file/{fileId}/preview | 预览归档文件 |

## 3. 归档文件管理

### 3.1 上传归档文件

**接口描述**: 上传归档文件到指定案件的指定分类

**请求信息**:
```
POST /api/v1/archive/{caseId}/upload
Content-Type: multipart/form-data
```

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| caseId | Long | 是 | 案件ID | 123 |

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| file | File | 是 | 文件对象 | - |
| categoryCode | String | 是 | 归档分类代码 | 0-1-1 |
| fileTitle | String | 否 | 文件标题 | 预重整启动文件 |
| fileDescription | String | 否 | 文件描述 | 预重整启动相关文件 |
| isConfidential | Boolean | 否 | 是否机密,默认false | false |
| accessLevel | String | 否 | 访问级别,默认INTERNAL | INTERNAL |

**支持的文件格式**: pdf, doc, docx, xls, xlsx, ppt, pptx, jpg, jpeg, png, gif, bmp, txt

**文件大小限制**: 单个文件不超过50MB

**响应示例**:
```json
{
  "code": 200,
  "message": "文件上传成功",
  "data": {
    "id": 1001,
    "caseId": 123,
    "categoryCode": "0-1-1",
    "fileId": 5001,
    "archiveNo": "AH-20260116-0001",
    "fileTitle": "预重整启动文件",
    "fileDescription": "预重整启动相关文件",
    "uploadUserId": 10,
    "uploadUserName": "张三",
    "uploadTime": "2026-01-16 10:00:00",
    "status": "ACTIVE",
    "isConfidential": false,
    "accessLevel": "INTERNAL",
    "version": 1,
    "parentVersionId": null,
    "file": {
      "id": 5001,
      "originalFileName": "预重整启动文件.pdf",
      "fileSize": 1048576,
      "fileExtension": "pdf",
      "mimeType": "application/pdf"
    }
  }
}
```

**字段说明**:
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 记录ID |
| caseId | Long | 案件ID |
| categoryCode | String | 归档分类代码 |
| categoryName | String | 归档分类名称 |
| fileId | Long | 文件ID |
| archiveNo | String | 归档编号 |
| fileTitle | String | 文件标题 |
| fileDescription | String | 文件描述 |
| uploadUserId | Long | 上传人ID |
| uploadUserName | String | 上传人姓名 |
| uploadTime | String | 上传时间 |
| status | String | 记录状态 |
| isConfidential | Boolean | 是否机密 |
| accessLevel | String | 访问级别 |
| version | Integer | 版本号 |
| parentVersionId | Long | 父版本ID |
| file | Object | 文件信息 |

**请求示例**:
```javascript
// 使用FormData上传文件
const formData = new FormData();
formData.append('file', file);
formData.append('categoryCode', '0-1-1');
formData.append('fileTitle', '预重整启动文件');
formData.append('fileDescription', '预重整启动相关文件');
formData.append('isConfidential', 'false');
formData.append('accessLevel', 'INTERNAL');

fetch('http://192.168.0.120:8080/api/v1/archive/123/upload', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer {token}'
  },
  body: formData
})
.then(response => response.json())
.then(data => console.log(data));
```

### 3.8 下载归档文件

**接口描述**: 下载归档文件

**请求信息**:
```
GET /api/v1/archive/file/{fileId}/download
```

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| fileId | Long | 是 | 文件ID | 5001 |

**响应**: 文件流(application/octet-stream)

**请求示例**:
```javascript
fetch('http://192.168.0.120:8080/api/v1/archive/file/5001/download', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer {token}'
  }
})
.then(response => response.blob())
.then(blob => {
  const url = window.URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = '预重整启动文件.pdf';
  document.body.appendChild(a);
  a.click();
  window.URL.revokeObjectURL(url);
  document.body.removeChild(a);
});
```

## 4. 通用说明

### 4.1 认证方式

所有接口都需要在请求头中携带认证Token:

```
Authorization: Bearer {token}
```

### 4.2 响应格式

所有接口统一使用以下响应格式:

**成功响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**错误响应**:
```json
{
  "code": 500,
  "message": "错误信息",
  "data": null
}
```

### 4.3 分页参数

分页接口统一使用以下参数:

| 参数名 | 类型 | 必填 | 说明 | 默认值 |
|--------|------|------|------|--------|
| pageNum | Integer | 否 | 页码 | 1 |
| pageSize | Integer | 否 | 每页大小 | 10 |

### 4.4 文件上传说明

**支持的文件格式**:
- 文档类: pdf, doc, docx, xls, xlsx, ppt, pptx, txt
- 图片类: jpg, jpeg, png, gif, bmp

**文件大小限制**: 单个文件不超过50MB

**上传方式**: 使用FormData进行multipart/form-data上传

### 4.5 访问级别说明

| 访问级别 | 说明 |
|----------|------|
| PUBLIC | 公开 |
| INTERNAL | 内部(默认) |
| CONFIDENTIAL | 机密 |
| TOP_SECRET | 绝密 |

### 4.6 记录状态说明

| 状态 | 说明 |
|------|------|
| ACTIVE | 激活(默认) |
| INACTIVE | 停用 |
| DELETED | 删除 |

### 4.7 归档分类说明

归档分类采用三级结构:

- **一级分类**: 6大类(0-5)
  - 0: 零、预重整/庭外重组阶段文件(若适用)
  - 1: 一、法院程序文件
  - 2: 二、管理人履职文件
  - 3: 三、债权人会议文件
  - 4: 四、资产处置文件
  - 5: 五、财务文件
  - 6: 六、案件终结与归档

- **二级分类**: 每个一级分类下的子分类
- **三级分类**: 具体的文件类型

## 5. 错误码说明

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 检查请求参数是否正确 |
| 401 | 未授权 | 检查Token是否有效 |
| 403 | 禁止访问 | 检查用户权限 |
| 404 | 资源不存在 | 检查资源ID是否正确 |
| 500 | 服务器错误 | 联系系统管理员 |

### 常见错误信息

| 错误信息 | 说明 | 处理建议 |

### B. 前端集成示例

#### B.1 Vue.js示例

```javascript
// api/archive.js
import request from '@/utils/request'

export function getCategoryTree(status) {
  return request({
    url: '/archive/categories',
    method: 'get',
    params: { status }
  })
}

export function uploadArchiveFile(caseId, data) {
  return request({
    url: `/archive/${caseId}/upload`,
    method: 'post',
    data: data,
    headers: {
      'Content-Type': 'multipart/form-data'
    }
  })
}

export function getArchiveFiles(caseId, params) {
  return request({
    url: `/archive/${caseId}/files`,
    method: 'get',
    params
  })
}

export function deleteArchiveRecord(recordId) {
  return request({
    url: `/archive/record/${recordId}`,
    method: 'delete'
  })
}
```

#### B.2 React示例

```javascript
// services/archiveService.js
import axios from 'axios';

const API_BASE_URL = 'http://192.168.0.120:8080/api/v1';

const archiveService = {
  getCategoryTree: async (status = 'ACTIVE') => {
    const response = await axios.get(`${API_BASE_URL}/archive/categories`, {
      params: { status }
    });
    return response.data;
  },

  uploadArchiveFile: async (caseId, formData) => {
    const response = await axios.post(
      `${API_BASE_URL}/archive/${caseId}/upload`,
      formData,
      {
        headers: {
          'Content-Type': 'multipart/form-data'
        }
      }
    );
    return response.data;
  },

  getArchiveFiles: async (caseId, params) => {
    const response = await axios.get(
      `${API_BASE_URL}/archive/${caseId}/files`,
      { params }
    );
    return response.data;
  },

  deleteArchiveRecord: async (recordId) => {
    const response = await axios.delete(
      `${API_BASE_URL}/archive/record/${recordId}`
    );
    return response.data;
  }
};

export default archiveService;
```
