## 1. 财产管理

### 1.1 创建财产

**接口名称**: 创建财产

**请求URL**: `/api/v1/property`

**请求方法**: POST

**请求参数**:

| 参数名 | 类型 | 必填 | 约束条件 | 说明 |
|--------|------|------|----------|------|
| caseId | Long | 是 | 非空 | 案件ID |
| caseName | String | 是 | 最大长度200 | 案件名称 |
| propertyType | String | 是 | 非空，最大长度50 | 财产类型: REAL_ESTATE-房产, VEHICLE-车辆, EQUIPMENT-设备, INVENTORY-存货, INTELLECTUAL_PROPERTY-知识产权, OTHER-其他 |
| propertyName | String | 是 | 非空，最大长度200 | 财产名称 |
| propertyDescription | String | 否 | TEXT类型 | 财产描述 |
| propertyLocation | String | 否 | 最大长度500 | 财产位置 |
| propertyAddress | String | 否 | 最大长度500 | 财产地址 |
| ownerName | String | 是 | 非空，最大长度200 | 所有权人名称 |
| ownerType | String | 否 | 最大长度50 | 所有权人类型: INDIVIDUAL-个人, ENTERPRISE-企业, OTHER-其他 |
| ownershipCertificate | String | 否 | 最大长度200 | 所有权证书 |
| certificateNo | String | 否 | 最大长度100 | 证书编号 |
| registrationDate | LocalDateTime | 否 | - | 登记日期 |
| acquisitionDate | LocalDateTime | 否 | - | 取得日期 |
| acquisitionMethod | String | 否 | 最大长度50 | 取得方式: PURCHASE-购买, DONATION-捐赠, INHERITANCE-继承, OTHER-其他 |
| acquisitionCost | BigDecimal | 否 | 精度18，小数位2 | 取得成本 |
| currentValue | BigDecimal | 否 | 精度18，小数位2 | 当前价值 |
| valuationDate | LocalDateTime | 否 | - | 评估日期 |
| valuationMethod | String | 否 | 最大长度50 | 评估方法: MARKET-市场法, COST-成本法, INCOME-收益法, OTHER-其他 |
| valuationInstitution | String | 否 | 最大长度200 | 评估机构 |
| valuationReport | String | 否 | 最大长度200 | 评估报告 |
| isEncumbered | Boolean | 否 | 默认false | 是否有权利负担 |
| encumbranceType | String | 否 | 最大长度50 | 权利负担类型: MORTGAGE-抵押, LIEN-留置, PLEDGE-质押, OTHER-其他 |
| encumbranceAmount | BigDecimal | 否 | 精度18，小数位2 | 权利负担金额 |
| encumbranceDate | LocalDateTime | 否 | - | 权利负担日期 |
| custodian | String | 否 | 最大长度100 | 保管人 |
| custodianContact | String | 否 | 最大长度100 | 保管人联系方式 |
| insuranceStatus | String | 否 | 最大长度20 | 保险状态: UNINSURED-未投保, INSURED-已投保, EXPIRED-已过期 |
| insuranceCompany | String | 否 | 最大长度200 | 保险公司 |
| insurancePolicyNo | String | 否 | 最大长度100 | 保险单号 |
| insuranceAmount | BigDecimal | 否 | 精度18，小数位2 | 保险金额 |
| insuranceStartDate | LocalDateTime | 否 | - | 保险开始日期 |
| insuranceEndDate | LocalDateTime | 否 | - | 保险结束日期 |
| relatedDebtId | Long | 否 | - | 关联债务ID |
| relatedClaimId | Long | 否 | - | 关联债权ID |
| attachments | String | 否 | TEXT类型 | 附件 |
| remarks | String | 否 | TEXT类型 | 备注 |

**响应格式**:

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "propertyId": 1,
    "propertyNo": "PROP20260112000001"
  }
}
```

**示例请求**:

```json
{
  "caseId": 1,
  "caseName": "某某公司破产案",
  "propertyType": "REAL_ESTATE",
  "propertyName": "某某房产",
  "propertyAddress": "北京市朝阳区某某街道",
  "ownerName": "某某公司",
  "ownerType": "ENTERPRISE",
  "acquisitionMethod": "PURCHASE",
  "acquisitionCost": 1000000.00,
  "currentValue": 1200000.00,
  "valuationMethod": "MARKET",
  "valuationInstitution": "某某评估机构",
  "isEncumbered": false,
  "insuranceStatus": "INSURED",
  "insuranceCompany": "某某保险公司"
```

## 10. 附录

### 10.1 注意事项

1. **数据完整性**: 创建资产时，系统会自动生成唯一编号
2. **状态流转**: 资产状态和管理状态的流转需要遵循业务规则
3. **权限控制**: 所有接口都需要登录认证，部分接口需要特定权限
4. **数据验证**: 所有接口都会对输入参数进行验证，不符合要求的请求会返回错误
5. **事务处理**: 涉及多个表的操作都在事务中执行，保证数据一致性
6. **资产关联**: 房产、车辆、设备、存货、知识产权可以关联到财产记录
7. **评估要求**: 资产价值评估需要由专业评估机构进行，并上传评估报告
8. **保险管理**: 建议为高价值资产购买保险，并记录保险信息

### 10.2 性能建议

1. **分页查询**: 列表接口建议使用分页查询，避免一次性查询大量数据
2. **索引优化**: 案件ID、资产ID等关键字段已建立索引
3. **缓存策略**: 常用数据可以考虑使用缓存提高查询效率
4. **批量操作**: 大量资产录入时建议使用批量导入功能

### 10.3 安全建议

1. **数据加密**: 敏感信息建议在传输和存储时进行加密
2. **访问控制**: 严格控制接口访问权限
3. **日志记录**: 重要操作需要记录操作日志，便于追溯
4. **数据备份**: 定期备份资产数据，防止数据丢失
