# 法律案件管理系统 - 项目总结

**日期**: 2026年1月23日
**版本**: v5.5.9
**技术栈**: Vue 3 + TypeScript + Vite + Element Plus

---

## 一、项目概述

本项目是一款专为律师事务所和破产管理人设计的**法律案件全流程管理系统**,基于Vue Vben Admin框架开发,采用最新的前端技术栈,提供从案件受理到结案归档的完整数字化解决方案。

### 核心价值
- **全流程数字化**: 覆盖破产案件从受理到结案的全部环节
- **协同办公**: 支持工作团队协作、实时通讯、任务分配
- **数据驱动**: 提供丰富的统计分析功能,辅助决策
- **权限管理**: 细粒度的权限控制,保障数据安全

---

## 二、系统架构

### 技术架构
```
前端层: Vue 3 + TypeScript + Element Plus
状态管理: Pinia
路由管理: Vue Router
实时通讯: WebSocket (STOMP协议)
图表可视化: ECharts
富文本编辑: WangEditor
文件处理: XLSX、文件上传下载
```

### 项目结构
```
vue-vben-admin-5.5.9/
├── apps/web-ele/           # 主应用
│   ├── src/
│   │   ├── api/           # API接口层
│   │   ├── views/         # 页面视图
│   │   ├── components/    # 公共组件
│   │   ├── router/        # 路由配置
│   │   └── store/         # 状态管理
├── packages/              # 共享包
└── internal/             # 内部工具
```

## 三、核心功能模块

### 1. 案件管理模块

**功能概述**: 案件信息的全生命周期管理

**主要功能**:
- 案件创建与编辑
- 案件状态流转 (待受理、进行中、已完成、已归档等)
- 案件进度管理 (七个阶段:FIRST到SEVENTH)
- 案件审核流程
- 案件查询与筛选

**核心字段**:
- 案件编号、案件名称
- 受理法院、指定机构
- 主办人员、承办人员
- 是否简易程序
- 案件来源、案由
- 债权申报期限
- 各关键日期 (受理日、立案日、破产日等)

**API端点**:
```
POST   /api/v1/case                    # 创建案件
GET    /api/v1/case/{caseId}           # 获取案件详情
PUT    /api/v1/case/{caseId}           # 更新案件
PUT    /api/v1/case/{caseId}/status    # 状态流转
PUT    /api/v1/case/{caseId}/progress  # 进度更新
POST   /api/v1/case/{caseId}/review    # 案件审核
GET    /api/v1/case/list               # 案件列表
GET    /api/v1/case/{caseId}/related-data  # 关联数据
```

### 2. 债权申报模块

**功能概述**: 债权人债权申报的全流程管理

**主要功能**:
- 债权申报登记 (分三个阶段)
  - 阶段一: 基本信息录入
  - 阶段二: 债权详情填写
  - 阶段三: 证据材料上传
- 债权审查
- 债权确认
- 材料接收与完整性检查
- 债权申报查询

**核心字段**:
- 债权人信息 (名称、类型、统一社会信用代码)
- 代理人信息
- 债权金额 (本金、利息、罚息、其他损失)
- 债权性质、债权类型
- 是否有判决、执行、担保
- 申报材料 (证据清单、证据材料、附件)
- 材料接收状态

**API端点**:
```
POST   /api/v1/claim-registration                    # 创建债权申报
GET    /api/v1/claim-registration/{claimId}          # 获取详情
GET    /api/v1/claim-registration/list              # 申报列表
PUT    /api/v1/claim-registration/{claimId}         # 更新申报
DELETE /api/v1/claim-registration/{claimId}         # 删除申报
PUT    /api/v1/claim-registration/{claimId}/status  # 更新状态
POST   /api/v1/claim-registration/{claimId}/material  # 接收材料
```
