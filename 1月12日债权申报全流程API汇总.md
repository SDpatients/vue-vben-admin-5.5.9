## 1. 债权申报登记管理

### 1.1 创建债权申报登记

**接口名称**: 创建债权申报登记

**请求URL**: `/api/v1/claim-registration`

**请求方法**: POST

**请求参数**:

| 参数名 | 类型 | 必填 | 约束条件 | 说明 |
|--------|------|------|----------|------|
| caseId | Long | 是 | 非空 | 案件ID |
| caseName | String | 否 | 最大长度255 | 案件名称 |
| debtor | String | 否 | 最大长度255 | 债务人 |
| creditorName | String | 是 | 非空，最大长度255 | 债权人名称 |
| creditorType | String | 是 | 非空，最大长度50 | 债权人类型 |
| creditCode | String | 否 | 最大长度100 | 信用代码 |
| legalRepresentative | String | 否 | 最大长度100 | 法定代表人 |
| serviceAddress | String | 否 | 最大长度500 | 送达地址 |
| agentName | String | 否 | 最大长度100 | 代理人姓名 |
| agentPhone | String | 否 | 最大长度50 | 代理人电话 |
| agentIdCard | String | 否 | 最大长度50 | 代理人身份证号 |
| agentAddress | String | 否 | 最大长度500 | 代理人地址 |
| accountName | String | 否 | 最大长度255 | 账户名称 |
| creditorBankAccount | String | 否 | 最大长度100 | 银行账号 |
| bankName | String | 否 | 最大长度255 | 开户银行 |
| principal | BigDecimal | 否 | 精度18，小数位2 | 申报本金 |
| interest | BigDecimal | 否 | 精度18，小数位2 | 申报利息 |
| penalty | BigDecimal | 否 | 精度18，小数位2 | 申报罚金 |
| otherLosses | BigDecimal | 否 | 精度18，小数位2 | 申报其他损失 |
| totalAmount | BigDecimal | 是 | 非空，精度18，小数位2 | 申报总金额 |
| hasCourtJudgment | Integer | 否 | 默认0 | 是否有法院判决: 0-否, 1-是 |
| hasExecution | Integer | 否 | 默认0 | 是否有执行: 0-否, 1-是 |
| hasCollateral | Integer | 否 | 默认0 | 是否有担保: 0-否, 1-是 |
| claimNature | String | 否 | 最大长度50 | 债权性质 |
| claimType | String | 是 | 非空，最大长度50 | 债权类型 |
| claimFacts | String | 否 | TEXT类型 | 债权事实 |
| claimIdentifier | String | 否 | 最大长度100 | 债权标识 |
| evidenceList | String | 否 | TEXT类型 | 证据清单 |
| evidenceMaterials | String | 否 | TEXT类型 | 证据材料 |
| evidenceAttachments | String | 否 | TEXT类型 | 证据附件 |
| registrationDate | LocalDateTime | 否 | - | 申报日期 |
| registrationDeadline | LocalDateTime | 否 | - | 申报截止日期 |
| materialReceiver | String | 否 | 最大长度100 | 材料接收人 |
| materialReceiveDate | LocalDateTime | 否 | - | 材料接收日期 |
| materialCompleteness | String | 否 | 最大长度50 | 材料完整性: COMPLETE-完整, INCOMPLETE-不完整, PENDING-待补充 |
| remarks | String | 否 | TEXT类型 | 备注 |

**响应格式**:

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "claimId": 1,
    "claimNo": "CLAIM20260112000001"
  }
}
```

## 2. 债权审查管理

### 2.1 创建债权审查记录

**接口名称**: 创建债权审查记录

**请求URL**: `/api/v1/claim-review`

**请求方法**: POST

**请求参数**:

| 参数名 | 类型 | 必填 | 约束条件 | 说明 |
|--------|------|------|----------|------|
| claimRegistrationId | Long | 是 | 非空 | 债权申报ID |
| caseId | Long | 否 | - | 案件ID |
| creditorName | String | 否 | 最大长度255 | 债权人名称 |
| reviewDate | LocalDateTime | 否 | - | 审查日期 |
| reviewer | String | 否 | 最大长度100 | 审查人 |
| reviewRound | Integer | 否 | 默认1 | 审查轮次 |
| reviewBasis | String | 否 | TEXT类型 | 审查依据 |
| declaredPrincipal | BigDecimal | 否 | 精度18，小数位2 | 申报本金 |
| declaredInterest | BigDecimal | 否 | 精度18，小数位2 | 申报利息 |
| declaredPenalty | BigDecimal | 否 | 精度18，小数位2 | 申报罚金 |
| declaredOtherLosses | BigDecimal | 否 | 精度18，小数位2 | 申报其他损失 |
| declaredTotalAmount | BigDecimal | 否 | 精度18，小数位2 | 申报总金额 |
| confirmedPrincipal | BigDecimal | 否 | 精度18，小数位2 | 确认本金 |
| confirmedInterest | BigDecimal | 否 | 精度18，小数位2 | 确认利息 |
| confirmedPenalty | BigDecimal | 否 | 精度18，小数位2 | 确认罚金 |
| confirmedOtherLosses | BigDecimal | 否 | 精度18，小数位2 | 确认其他损失 |
| confirmedTotalAmount | BigDecimal | 否 | 精度18，小数位2 | 确认总金额 |
| unconfirmedPrincipal | BigDecimal | 否 | 精度18，小数位2 | 不予确认本金 |
| unconfirmedInterest | BigDecimal | 否 | 精度18，小数位2 | 不予确认利息 |
| unconfirmedPenalty | BigDecimal | 否 | 精度18，小数位2 | 不予确认罚金 |
| unconfirmedOtherLosses | BigDecimal | 否 | 精度18，小数位2 | 不予确认其他损失 |
| unconfirmedTotalAmount | BigDecimal | 否 | 精度18，小数位2 | 不予确认总金额 |
| adjustmentReason | String | 否 | TEXT类型 | 金额调整理由 |
| unconfirmedReason | String | 否 | TEXT类型 | 不予确认理由 |
| insufficientEvidenceReason | String | 否 | TEXT类型 | 证据不足理由 |
| expiredReason | String | 否 | TEXT类型 | 超过诉讼时效理由 |
| evidenceAuthenticity | String | 否 | 最大长度20 | 证据真实性: AUTHENTIC-真实, SUSPICIOUS-可疑, FAKE-虚假 |
| evidenceRelevance | String | 否 | 最大长度20 | 证据关联性: RELEVANT-相关, IRRELEVANT-不相关 |
| evidenceLegality | String | 否 | 最大长度20 | 证据合法性: LEGAL-合法, ILLEGAL-不合法 |
| evidenceReviewNotes | String | 否 | TEXT类型 | 证据审查说明 |
| confirmedClaimNature | String | 否 | 最大长度50 | 确认债权性质 |
| isJointLiability | Integer | 否 | 默认0 | 是否连带责任: 0-否, 1-是 |
| isConditional | Integer | 否 | 默认0 | 是否附条件: 0-否, 1-是 |
| isTerm | Integer | 否 | 默认0 | 是否附期限: 0-否, 1-是 |
| collateralType | String | 否 | 最大长度50 | 担保方式 |
| collateralProperty | String | 否 | 最大长度255 | 担保财产 |
| collateralAmount | BigDecimal | 否 | 精度18，小数位2 | 担保金额 |
| collateralTerm | String | 否 | 最大长度100 | 担保期限 |
| collateralValidity | String | 否 | 最大长度20 | 担保有效性: VALID-有效, INVALID-无效, PARTIAL-部分有效 |
| reviewConclusion | String | 否 | 最大长度20 | 审查结论: CONFIRMED-确认, PARTIAL_CONFIRMED-部分确认, UNCONFIRMED-不予确认 |
| reviewSummary | String | 否 | TEXT类型 | 审查总结 |
| reviewReport | String | 否 | TEXT类型 | 审查报告 |
| reviewAttachments | String | 否 | TEXT类型 | 审查附件 |
| reviewStatus | String | 否 | 默认COMPLETED | 审查状态: PENDING-待审查, IN_PROGRESS-审查中, COMPLETED-已完成, SUPPLEMENT-待补充 |
| remarks | String | 否 | TEXT类型 | 备注 |

**响应格式**:

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "reviewId": 1,
    "reviewRound": 1
  }
}
```

```
2. 管理人接收申报材料
   ↓
3. 管理人登记债权申报信息
   ↓
4. 系统自动创建待审查记录和待确认记录
```

**相关API接口**:
- 创建债权申报登记: `POST /api/v1/claim-registration`
- 接收申报材料: `POST /api/v1/claim-registration/{claimId}/material`
- 更新债权申报状态: `PUT /api/v1/claim-registration/{claimId}/status`

---

### 5.2 债权审查流程

```
1. 管理人查看待审查债权
   ↓
2. 管理人进行债权审查
   ↓
3. 填写审查信息和审查结论
   ↓
4. 提交审查结果
   ↓
5. 如需补充材料，通知债权人补充
   ↓
6. 完成审查
```

**相关API接口**:
- 获取待审查的债权: `GET /api/v1/claim-review/pending/{caseId}`
- 创建债权审查记录: `POST /api/v1/claim-review`
- 更新债权审查记录: `PUT /api/v1/claim-review/{reviewId}`
- 提交债权审查: `POST /api/v1/claim-review/{reviewId}/submit`

---

### 5.3 债权确认流程

```
1. 召开债权人会议
   ↓
2. 进行债权表决
   ↓
3. 记录表决结果
   ↓
4. 如有异议，进入异议处理流程
   ↓
5. 协商或法院裁定
   ↓
6. 最终确认债权
```

**相关API接口**:
- 提交债权人会议表决: `POST /api/v1/claim-confirmation/{confirmationId}/vote`
- 提交债权异议: `POST /api/v1/claim-confirmation/{confirmationId}/objection`
- 处理异议协商: `POST /api/v1/claim-confirmation/{confirmationId}/negotiation`
- 提交法院裁定: `POST /api/v1/claim-confirmation/{confirmationId}/court-ruling`
- 最终确认债权: `POST /api/v1/claim-confirmation/{confirmationId}/finalize`

---

### 5.4 异议之诉流程

```
1. 债权人或管理人提起诉讼
   ↓
2. 记录诉讼信息
   ↓
3. 更新诉讼状态
   ↓
4. 法院判决后记录判决结果
   ↓
5. 根据判决结果最终确认债权
```

**相关API接口**:
- 更新债权确认记录: `PUT /api/v1/claim-confirmation/{confirmationId}`
- 更新诉讼状态: `PUT /api/v1/claim-confirmation/{confirmationId}/lawsuit-status`
- 最终确认债权: `POST /api/v1/claim-confirmation/{confirmationId}/finalize`

## 7. 附录

### 7.1 注意事项

1. **数据完整性**: 创建债权申报时，系统会自动创建对应的审查记录和确认记录
2. **状态流转**: 债权申报、审查、确认的状态流转需要遵循业务规则
3. **权限控制**: 所有接口都需要登录认证，部分接口需要特定权限
4. **数据验证**: 所有接口都会对输入参数进行验证，不符合要求的请求会返回错误
5. **事务处理**: 涉及多个表的操作都在事务中执行，保证数据一致性

### 7.2 性能建议

1. **分页查询**: 列表接口建议使用分页查询，避免一次性查询大量数据
2. **索引优化**: 案件ID、债权申报ID等关键字段已建立索引
3. **缓存策略**: 常用数据可以考虑使用缓存提高查询效率

### 7.3 安全建议

1. **数据加密**: 敏感信息建议在传输和存储时进行加密
2. **访问控制**: 严格控制接口访问权限
3. **日志记录**: 重要操作需要记录操作日志，便于追溯
