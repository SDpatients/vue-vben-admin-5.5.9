# 债权人会议数据库表结构

## 1. 创建数据库（如果需要）

```sql
CREATE DATABASE IF NOT EXISTS bankruptcy_meeting DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE bankruptcy_meeting;
```

## 2. 创建表结构

### 2.1 投票项表（meeting_vote_items）

```sql
CREATE TABLE IF NOT EXISTS meeting_vote_items (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '投票项ID',
    meeting_id INT NOT NULL COMMENT '会议ID，关联会议表',
    item_name VARCHAR(255) NOT NULL COMMENT '投票项名称',
    agree_count INT DEFAULT 0 COMMENT '同意票数',
    oppose_count INT DEFAULT 0 COMMENT '反对票数',
    abstain_count INT DEFAULT 0 COMMENT '弃权票数',
    remark VARCHAR(255) COMMENT '备注',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) COMMENT='债权人会议投票项表';
```

### 2.2 视频标签表（meeting_video_tags）

```sql
CREATE TABLE IF NOT EXISTS meeting_video_tags (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '视频标签ID',
    meeting_id INT NOT NULL COMMENT '会议ID，关联会议表',
    video_title VARCHAR(255) NOT NULL COMMENT '视频标题',
    status ENUM('generated', 'pending') DEFAULT 'pending' COMMENT '生成状态：generated=已生成，pending=待生成',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) COMMENT='债权人会议视频标签表';
```

## 4. 表结构说明

### 4.1 meeting_vote_items表
- **存储内容**：债权人会议的投票项信息
- **核心字段**：
  - `item_name`：投票项名称
  - `agree_count`：同意票数
  - `oppose_count`：反对票数
  - `abstain_count`：弃权票数
  - `remark`：备注（如"已通过"、"待表决"等）
- **时间字段**：自动记录创建和更新时间

### 4.2 meeting_video_tags表
- **存储内容**：债权人会议的视频标签信息
- **核心字段**：
  - `video_title`：视频标题
  - `status`：生成状态（已生成/待生成）
- **时间字段**：自动记录创建和更新时间

## 5. 数据说明

- **投票项数据**：包含5个典型的债权人会议投票项，涵盖财产方案、委员会选举等关键议题
- **视频标签数据**：包含6个会议视频标签，覆盖会议的主要环节
- **状态多样性**：投票项包含"已通过"和"待表决"状态，视频标签包含"已生成"和"待生成"状态

## 6. 使用建议

1. **与会议表关联**：实际使用时，建议创建会议表（meeting），通过meeting_id字段与上述表关联
2. **索引优化**：对于大数据量场景，建议在meeting_id字段上创建索引
3. **权限控制**：根据实际业务需求，添加适当的权限控制字段
4. **扩展字段**：可根据具体业务需求，适当扩展表结构字段

此表结构设计合理，字段齐全，注释清晰，能够很好地支持债权人会议数据管理需求。
