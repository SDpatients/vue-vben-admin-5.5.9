## 1. 债权申报登记管理

### 基础路径
`/claim-registration`

### 1.1 创建债权申报登记

**接口描述**：创建新的债权申报登记记录

**请求方式**：`POST`

**请求路径**：`/claim-registration`

**请求参数**：
```json
{
  "caseId": 1,                              // 必填，案件ID
  "caseName": "某某公司破产案",              // 案件名称
  "debtor": "某某公司",                      // 债务人

  "creditorName": "债权人A",                 // 必填，债权人名称
  "creditorType": "企业",                   // 必填，债权人类型
  "creditCode": "91110000XXXXXXXXXX",       // 统一社会信用代码
  "legalRepresentative": "张三",            // 法定代表人
  "serviceAddress": "北京市朝阳区...",       // 送达地址
  "agentName": "李四",                      // 代理人姓名
  "agentPhone": "13800138000",              // 代理人电话
  "agentIdCard": "110101199001011234",      // 代理人身份证
  "agentAddress": "北京市海淀区...",         // 代理人地址
  "accountName": "债权人A",                 // 账户名称
  "creditorBankAccount": "6222020200000000", // 债权人银行账号
  "bankName": "中国工商银行",               // 开户银行
  "principal": 1000000.00,                  // 本金
  "interest": 50000.00,                     // 利息
  "penalty": 10000.00,                      // 违约金
  "otherLosses": 5000.00,                   // 其他损失
  "totalAmount": 1065000.00,                // 必填，总金额
  "hasCourtJudgment": 0,                    // 是否有法院判决书（0：否，1：是）
  "hasExecution": 0,                        // 是否有执行文书（0：否，1：是）
  "hasCollateral": 0,                       // 是否有担保（0：否，1：是）
  "claimNature": "有担保债权",               // 债权性质
  "claimType": "普通债权",                   // 必填，债权类型
  "claimFacts": "债权事实描述...",           // 债权事实
  "claimIdentifier": "债权编号",             // 债权标识
  "evidenceList": "证据清单",               // 证据清单
  "evidenceMaterials": "证据材料",           // 证据材料
  "evidenceAttachments": "附件路径",         // 证据附件
  "registrationDate": "2026-01-15T10:00:00", // 登记日期
  "registrationDeadline": "2026-02-15T10:00:00", // 登记截止日期
  "materialReceiver": "接收人",              // 材料接收人
  "materialReceiveDate": "2026-01-15T10:00:00", // 材料接收日期
  "materialCompleteness": "完整",           // 材料完整性
  "remarks": "备注信息"                      // 备注
}
```

**返回结果**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "claimId": 1,
    "claimNo": "CL202601150001"
  }
}
```

## 2. 债权审查管理

### 基础路径
`/claim-review`

### 2.1 创建债权审查记录

**接口描述**：创建新的债权审查记录

**请求方式**：`POST`

**请求路径**：`/claim-review`

**请求参数**：
```json
{
  "claimRegistrationId": 1,                 // 必填，债权申报ID
  "caseId": 1,                              // 案件ID
  "creditorName": "债权人A",                 // 债权人名称
  "reviewDate": "2026-01-16T10:00:00",     // 审查日期
  "reviewer": "审查人",                     // 审查人
  "reviewRound": 1,                         // 审查轮次（默认：1）
  "reviewBasis": "审查依据",                // 审查依据
  "declaredPrincipal": 1000000.00,         // 申报本金
  "declaredInterest": 50000.00,             // 申报利息
  "declaredPenalty": 10000.00,              // 申报违约金
  "declaredOtherLosses": 5000.00,           // 申报其他损失
  "declaredTotalAmount": 1065000.00,       // 申报总金额
  "confirmedPrincipal": 1000000.00,         // 确认本金
  "confirmedInterest": 50000.00,            // 确认利息
  "confirmedPenalty": 10000.00,             // 确认违约金
  "confirmedOtherLosses": 5000.00,          // 确认其他损失
  "confirmedTotalAmount": 1065000.00,       // 确认总金额
  "unconfirmedPrincipal": 0.00,             // 未确认本金
  "unconfirmedInterest": 0.00,              // 未确认利息
  "unconfirmedPenalty": 0.00,               // 未确认违约金
  "unconfirmedOtherLosses": 0.00,          // 未确认其他损失
  "unconfirmedTotalAmount": 0.00,          // 未确认总金额
  "adjustmentReason": "调整原因",           // 调整原因
  "unconfirmedReason": "未确认原因",        // 未确认原因
  "insufficientEvidenceReason": "证据不足原因", // 证据不足原因
  "expiredReason": "过期原因",              // 过期原因
  "evidenceAuthenticity": "真实",           // 证据真实性
  "evidenceRelevance": "相关",              // 证据相关性
  "evidenceLegality": "合法",                // 证据合法性
  "evidenceReviewNotes": "证据审查备注",   // 证据审查备注
  "confirmedClaimNature": "有担保债权",     // 确认债权性质
  "isJointLiability": 0,                    // 是否连带责任（0：否，1：是）
  "isConditional": 0,                       // 是否附条件（0：否，1：是）
  "isTerm": 0,                              // 是否附期限（0：否，1：是）
  "collateralType": "抵押",                 // 担保类型
  "collateralProperty": "房产",             // 担保财产
  "collateralAmount": 500000.00,            // 担保金额
  "collateralTerm": "担保期限",              // 担保期限
  "collateralValidity": "有效",             // 担保有效性
  "reviewConclusion": "审查结论",           // 审查结论
  "reviewSummary": "审查摘要",              // 审查摘要
  "reviewReport": "审查报告",               // 审查报告
  "reviewAttachments": "附件路径",          // 审查附件
  "reviewStatus": "PENDING",                // 审查状态（默认：PENDING）

}
```

## 7. 全流程说明

### 7.1 债权登记表全流程

1. **债权申报登记阶段**
   - 创建债权申报登记（POST /claim-registration）
   - 接收申报材料（POST /claim-registration/{claimId}/material）
   - 更新债权申报状态（PUT /claim-registration/{claimId}/status）

2. **债权审查阶段**
   - 创建债权审查记录（POST /claim-review）
   - 更新债权审查记录（PUT /claim-review/{reviewId}）
   - 提交债权审查（POST /claim-review/{reviewId}/submit）
   - 查看待审查债权（GET /claim-review/pending/{caseId}）

3. **债权确认与异议阶段**
   - 创建债权确认记录（POST /claim-confirmation）
   - 提交债权人会议表决（POST /claim-confirmation/{confirmationId}/vote）
   - 如有异议，提交债权异议（POST /claim-confirmation/{confirmationId}/objection）
   - 处理异议协商（POST /claim-confirmation/{confirmationId}/negotiation）
   - 如协商不成，提交法院裁定（POST /claim-confirmation/{confirmationId}/court-ruling）
   - 如有诉讼，更新诉讼状态（PUT /claim-confirmation/{confirmationId}/lawsuit-status）
   - 最终确认债权（POST /claim-confirmation/{confirmationId}/finalize）

4. **债权申报审核阶段**
   - 债权申报审核（POST /creditor-claim/{claimId}/review）
   - 查询审核状态（GET /creditor-claim/{claimId}/review-status）

5. **统计分析阶段**
   - 获取债权申报统计数据（GET /creditor-claim-statistics）
   - 获取审查统计（GET /claim-review/statistics/{caseId}）
   - 获取确认统计（GET /claim-confirmation/statistics/{caseId}）

### 7.2 状态流转

**债权申报登记状态流转**：
```
PENDING（待处理） → REGISTERED（已登记） → COMPLETED（已完成）
                ↓
              REJECTED（已拒绝）
```

**债权审查状态流转**：
```
PENDING（待审查） → IN_PROGRESS（审查中） → COMPLETED（已完成）
                ↓
              SUPPLEMENT（需补充材料） → IN_PROGRESS（审查中）
```

**债权确认状态流转**：
```
PENDING（待确认） → CONFIRMED（已确认）
                ↓
              OBJECTION（有异议） → COURT（法院裁定中） → CONFIRMED（已确认）
                                        ↓
                                      LAWSUIT（诉讼中） → CONFIRMED（已确认）
```

## 8. 注意事项

1. **日期格式**：所有日期字段使用ISO 8601格式（YYYY-MM-DDTHH:mm:ss）
2. **金额格式**：所有金额字段使用BigDecimal类型，保留两位小数
3. **分页参数**：pageNum从1开始，pageSize默认为10
4. **必填字段**：请参考各接口的请求参数说明，必填字段必须提供
5. **状态枚举**：请参考数据字典中的状态枚举值
6. **权限控制**：部分接口可能需要特定权限才能访问
7. **错误处理**：所有接口返回统一的错误格式，包含code、message、data三个字段
